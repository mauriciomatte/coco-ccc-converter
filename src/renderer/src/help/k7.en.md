# K7 Tab — Cassette Tape

The **K7** tab reads, tunes, edits, decodes and converts CoCo/Dragon **cassette tapes**. It is where you
**recover** programs from old tapes, **preserve** a tape as a clean file, **convert** loader games, and
**test** everything in an embedded mini-emulator. By the end of this guide you will know which button to use
in each situation and what each one delivers.

---

## 1. What you can open
- **WAV** — the digitized tape audio (what you capture from a recorder).
- **CAS / C10** — the digital image (the tape bytes) used by emulators.
- **VOC** — old audio (Creative Voice File).

Use **Open** or **drag** the file onto the waveform area. When you open a `.cas/.c10`, the app already turns
on **"Fast read"** and zooms into a small window so you can see the FSK square waves.

---

## 2. The waveform

The waveform is drawn with a **time ruler**, a yellow **playhead** and zoom. A corner button cycles the
**drawing style**:
- **line** — oscilloscope; the only one that shows the **real shape** (the FSK square wave). Use it zoomed in.
- **bars** (default), **peak** (filled envelope), **RMS** (energy) — good for viewing the whole tape.

**Zoom:** mouse wheel (centered on the cursor), zoom in/out buttons, and a **scrollbar** that appears when
zoomed. **Selection:** drag over the waveform to mark a span (highlighted) — that selection feeds all
editing. A single click positions the playhead (seek).

---

## 3. The player and the animated cassette

Below the waveform is an **animated datacorder**: the reels spin during playback. The controls:
**Rewind · Play/Pause · ● REC · Stop · Fast-forward · Eject**, a time **counter** and the **Speed**
selector:
- **Epoch** — the CoCo's real speed **measured** from the file itself (shows the factor, e.g. "1.03×"). The
  recommended one.
- **Normal** (1×), **-2× / -4×** (slows down, helps inspection), **×2 / ×4** (speeds up).

It is all monitoring — it **does not change the data**.

---

## 4. Decode — see the tape's programs

On open, the app **demodulates the FSK** automatically and shows, in the **"Program"** panel (right column):
**name, load, exec, type** (BASIC/Machine/Data) and the size. The indicator shows **✓ sync · N files**
(green) or **✗ no sync** (yellow). Expand the panel for all fields (rate, format, length, the program's RAM
class). If the tape holds several programs, a **list** appears — click each to extract that one.

> **Incomplete decode?** If there is far more audio than recovered data, a yellow warning appears: the tape
> probably has a **screen + loader** (section 7) or is degraded (section 5).

---

## 5. Sound adjustments ≠ Recover — the biggest confusion

In the **"Sound adjustments"** panel:
- **Threshold (µs)** — the point separating bit 0 from bit 1 (default 600). **Min amplitude** — ignores
  weak noise. **Default** — resets everything to safe (Threshold 600, Amplitude 0, WAV 22 kHz). → Use for
  **MANUAL fine tuning** when the tape almost syncs.
- **Recover** (✨) — turns on the **AUTOMATIC, adaptive** decode: it calibrates the threshold **per segment**
  (Otsu method + sweep) and ignores the manual settings. → Use on **degraded tapes** (40+ years, slow,
  stretched) that give "NO-SYNC".

**Rule of thumb:** first enable **Recover**; if still incomplete, tweak **Threshold/Amplitude**; if it gets
worse, click **Default** and start over. The app re-decodes on every change.

---

## 6. The live panels: BASIC · HEX · mini-XRoar

Below the player are three panels (adjustable widths):
- **BASIC (detokenized · 32 col)** — shows the BASIC program already **detokenized**, wrapped to 32 columns
  (the CoCo's screen width). During **Play**, the text is revealed "typewriter" style, synced to the
  playhead — you watch the program "load" like back in the day.
- **Hex (program)** — a hex+ASCII dump of the full **raw stream**, also revealed byte by byte as the tape
  plays (header → screen/loader → turbo game).
- **mini-XRoar** — an embedded **muted** mini-emulator. Turn it on with the side button; when you **Play**,
  it resets and runs **CLOAD/CLOADM** on a small screen alongside, mirroring the load. The **"Fast read"**
  toggle: ON loads the `.cas` quickly (digital files); OFF plays the WAV in real time (with the real gaps,
  ideal for loader tapes).

---

## 7. Tape types — and what to do with each
- **STANDARD tape** (BASIC or ML saved normally): decodes fully → use the export/send buttons.
- **Tape with a SPLASH SCREEN + LOADER** (softhouse games, e.g. SoftKristian): the standard decoder reads
  only the namefile + the loader; the game comes after in a **turbo/custom stream** it doesn't decode (hence
  "incomplete decode"). For these, tweaking the sound won't help — use:
  - **"→ No loader"** (build an autostart BIN/DSK, keeping the screen), or
  - **"→ Full tape"** + run in **XRoar** (loads EVERYTHING like the original tape).

---

## 8. The conversion/export buttons — what each delivers

In the **"Export"** panel there is a **WAV kHz** selector (8/11/22/44) and a **real-time size preview**
(→ CAS, → WAV, → Full tape, original) so you can compare before saving.

| Button | What it does | When to use |
|---|---|---|
| **→ CAS** | **Normalize**: writes a CLEAN, tiny `.CAS` with just the decoded data | keep a reliable, small copy of a standard tape |
| **→ WAV** | **Normalize**: writes a CLEAN `.WAV` (≈11 kHz), much smaller, reliable on hardware | play on a real recorder/hardware |
| **→ Full tape** | writes the `.WAV` of the **entire ORIGINAL audio** — header, **screen**, loader and turbo game (22 kHz floor) | tapes with a splash screen that must run fully in XRoar |
| **→ No loader** | opens the **autostart** dialog (LOADM, no `:EXEC`) — removes the tape loader and keeps the screen | turn a loader-tape game into a disk file that runs by itself |
| **Revert** | opens one of OUR `.BIN` files (**CoCo DCU** signature) and recovers the **PURE PROGRAM** as `.CAS`/`.WAV` | undo a "→ No loader" and get just the program |
| **(→ DSK)** | writes the read file (BASIC/ML) into a **DSK pane** | put the tape's content on a disk |
| **(→ XRoar)** | loads the current tape into the **emulator** (XRoar tab) | test the tape right away |
| **(Open in BASIC)** | sends the tape's **BASIC** to the **BASIC editor** (detokenizes) | edit the BASIC program |
| **Extract** (Program panel) | saves the selected program in the chosen format (**.bin / .bas / .cas**) | pull one specific program off the tape |

> In short: **→ CAS/→ WAV = clean/normalized** (standard part only). **→ Full tape = everything, raw** (for
> screens/loaders). **→ No loader = convert to an autostart disk**. **Revert = the way back**.

### The "→ No loader (autostart)" dialog
It shows the detected loader (name, confidence, whether it has a 512B screen → $0400) and lets you adjust the
program's **Load** and **Exec**, choose the start trigger (**key** or **~10s delay**) and **autostart $0176**.
Yellow warnings flag if the program touches the DOS stack or reserved RAM. Buttons: **Write to DSK** (inject
into a pane) or **Save .BIN** (runs with LOADM, no `:EXEC`).

---

## 9. Editing the waveform

With a span **selected**: **cut / copy / paste / delete / crop-to-selection / normalize amplitude / insert
a pause (1.0 s of silence)**. There is **Undo / Redo**. Useful to trim dead leader/tail, split segments or
create gaps between programs. (Normalizing the amplitude often improves decoding.)

---

## 10. Recording from a real recorder (REC)

The **● REC** button **captures** the datacorder's audio via the PC's audio input (line-in): a **● REC**
indicator with a **VU** (level) appears and the reels spin. Click **Stop** to finish — the audio becomes a
waveform you decode/edit normally. If a tape is already open, the app asks before recording over it.

---

## 11. Typical flows (quick guide)
- **Standard tape, want to preserve it:** open → (Recover if needed) → **→ CAS** (or **→ WAV**).
- **Tape won't sync:** enable **Recover** → adjust **Threshold/Amplitude** → if worse, **Default**.
- **Game with screen + loader (SoftKristian):** open → **→ No loader** (builds autostart BIN/DSK) → test.
- **Screen game that won't convert:** **→ Full tape** → **→ XRoar**.
- **Want the pure program back:** open our `.BIN` with **Revert**.
- **Capture a physical tape:** connect the recorder to line-in → **● REC** → **Stop** → decode → export.
