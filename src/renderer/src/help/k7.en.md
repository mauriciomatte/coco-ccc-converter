# K7 Tab — Cassette Tape

The **K7** tab reads, tunes, edits and converts CoCo/Dragon **cassette tapes**. This help is written to
make the **workflow clear** — especially the difference between **adjusting the sound**, **recovering**
bad tapes, and the several **conversion/export** buttons (which are confusing at first).

## 1. What you can open
- **WAV** — the digitized tape audio.  **CAS** — the digital image (tape bytes).  **VOC** — old audio.
- Use **Open** or **drag** the file onto the waveform area.

## 2. The waveform, player and animated cassette
The waveform is drawn with a time ruler and zoom. There is **play/stop**, an **animated cassette** and a
**Speed** selector ("Epoch" = the CoCo's real speed analyzed from the tape; Normal; ×2/×4). It is purely
visual/monitoring — it does not change the data.

## 3. Decode (see the programs)
On open, the app **demodulates the FSK** and lists the tape's **programs**: **name, load, exec, type**
(BASIC/Machine/Data) and the **blocks** state (checksum). A tape may hold several.

## 4. Sound adjustments  ≠  Recover  (the biggest confusion)
In the **"Sound adjustments"** panel:
- **Threshold (µs)** — the point separating bit 0 from bit 1. **Min amplitude** — ignores weak noise.
  **kHz (WAV)** — exported WAV rate.  **Default** — resets everything to safe (Threshold 600, Amp 0,
  22 kHz).  → Use for **MANUAL fine tuning** when the tape almost syncs.
- **Recover** (✨) — enables an **AUTOMATIC, adaptive** decode (tries several thresholds, including one
  computed by Otsu) and keeps the result with the **most valid blocks**.  → Use for **degraded tapes**
  (40+ years, slow/stretched) that give "NO-SYNC".

**Rule of thumb:** first enable **Recover**; if still incomplete, tweak **Threshold/Amplitude**; if it
gets worse, click **Default** and start over.

## 5. Tape types — and what to do with each
- **STANDARD tape** (BASIC or ML saved normally): decodes fully → use the export/send buttons.
- **Tape with a SPLASH SCREEN + LOADER** (softhouse games, e.g. SoftKristian): the standard decoder
  **reads only the namefile + the loader**; the game comes after in a **turbo/custom stream** it doesn't
  decode (hence "incomplete decode"). For these, adjusting the sound won't help — use:
  - **"→ No loader"** (convert to an autostart BIN/DSK, keeping the screen), or
  - **"→ Full tape"** + run in **XRoar** (loads EVERYTHING like the original tape).

## 6. The conversion/export buttons (the workflow)
| Button | What it does | When to use |
|---|---|---|
| **→ CAS** | **Normalize**: writes a CLEAN, tiny `.CAS` with just the decoded data (standard CoCo/Dragon) | keep a reliable, small copy of a standard tape |
| **→ WAV** | **Normalize**: writes a CLEAN `.WAV` (e.g. 11 kHz), much smaller, reliable on real hardware | play on a real recorder / hardware |
| **→ Full tape** | writes the `.WAV` of the **entire ORIGINAL audio** — captures header, **screen**, loader and turbo game | tapes with a splash screen that must run fully in XRoar |
| **→ No loader** | SoftKristian tapes (screen+loader): build a `.BIN`/`.DSK` that **autostarts with LOADM** (no `:EXEC`) — **removes the tape loader and keeps the screen** | turn a loader-tape game into a disk file that runs by itself |
| **Revert** | opens one of OUR `.BIN` files (**CoCo DCU** signature) and recovers the **PURE PROGRAM** as `.CAS`/`.WAV` (without the screen/disk stub) | undo a "→ No loader" conversion and get just the program |
| **→ DSK** | writes the read file (BASIC/ML) into a **DSK pane** | put the tape's content on a disk |
| **→ XRoar** | loads the current tape into the **emulator** | test the tape right away |
| **Open in BASIC** | sends the tape's **BASIC** to the **BASIC editor** (detokenizes) | edit the BASIC program |

> Clearing up the common confusion: **"→ CAS/→ WAV" = clean/normalized** (standard part only). **"→ Full
> tape" = everything, raw** (for screens/loaders). **"→ No loader" = convert to an autostart disk.**
> **"Revert" = the way back** (recovers the pure program from our BIN).

## 7. Editing the waveform
**Select** a region and: **cut / copy / paste / delete / crop-to-selection / normalize amplitude /
insert a pause (silence)**. There is **Undo / Redo**. Useful to trim dead leader/tail or split segments.

## 8. Recording from a real recorder (REC)
The **● REC** indicator and the transport let you **capture** the datacorder's audio via the PC's audio
input (line-in) and then decode/edit it normally.

## 9. mini-XRoar and "Fast read"
There is an embedded mini-emulator to watch the load. The **Fast read** toggle: ON loads the `.cas`
quickly (digital files); OFF plays the WAV in real time (with the real gaps, for loader tapes).

## 10. Typical flows (quick guide)
- **Standard tape, want to preserve it:** open → (Recover if needed) → **→ CAS** (or **→ WAV**).
- **Tape won't sync:** enable **Recover** → adjust **Threshold/Amplitude** → if worse, **Default**.
- **Game with screen + loader (SoftKristian):** open → **→ No loader** (builds autostart BIN/DSK) → test.
- **Screen game that won't convert:** **→ Full tape** → **→ XRoar**.
- **Wrong conversion, want the pure program back:** open our `.BIN` with **Revert**.
