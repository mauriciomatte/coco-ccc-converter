v1.0.32 — 2026-06-08
--------------------------------------------------------------------------------
NEW — SDF (CoCoSDC) image reading — opens protected / mixed-density (FLEX) disks
  • SDF is the CoCoSDC format (Darren Atkinson) for non-standard / copy-protected disks -- a "pre-indexed
    DMK" (each track has a Sector ID Table in its header). The app now DECODES SDF to a raw sector image
    ON OPEN and proceeds normally (list/extract/edit), just like DMK. Works for RS-DOS, OS-9/RBF and
    Dragon. '.sdf' added to the "Open image" and "Open OS-9" dialogs.
  • False-positive-proof detection by the 'SDF1' MAGIC + exact size match (512 + cyl*sides*6656) -- never
    by extension ('.sdf' clashes with the SAM Coupe "Sam Disk Format"). Handles single density (FM, byte
    doubling), interleave, and flags protected sectors (bad CRC / deleted data mark).
  • Validated against a REAL sample (FHL Color FLEX 5.0.4): the .sdf decode matches 100% (630/630 sectors)
    against the SAME disk in DMK form. Read-only (generating SDF isn't needed for current flows).
  • The full SDF format spec is documented in docs/OS9_STATUS_EN.txt (section 7), from a technical study.

v1.0.27 — 2026-06-06
--------------------------------------------------------------------------------
NEW — "Recover" button (K7 tab): rescues degraded tapes that give NO-SYNC
  • 40+ year-old tapes (and imperfect captures) are often SLOW/stretched: the whole FSK is lengthened, so
    the fixed 600 µs threshold misreads the bits → NO-SYNC even when the data is intact. The new "Recover"
    button auto-CALIBRATES the 1/0 threshold, per segment, instead of assuming a fixed speed.
  • How it works: for EACH tape segment it measures the histogram of wave periods and finds the threshold
    that best separates the two groups (2400 Hz = bit 1 / 1200 Hz = bit 0) via OTSU; it also tries a
    threshold sweep (550–775 µs) with and without an amplitude (hiss) gate, and keeps the setting that
    recovers the MOST valid blocks (checksum OK). It auto-follows the tape's real speed (e.g. ~730 µs
    instead of 600).
  • Toggle in the K7 tab (✨ icon); when on, it ignores the manual Threshold/Amplitude and re-decodes by
    itself. The continuous fallback (tapes with no clear gaps) now sweeps thresholds too.
  • First step of the recovery ROADMAP (R1): unlocks slow/stretched tapes. Planned next: visual diagnostics
    (good/bad block map) and fusing multiple captures ("tape RAID").

FIX — "Revert"/"→ No loader": .WAV now loads with the standard reader
  • The .WAV produced when recovering the pure program (Revert / loader-strip) was built with almost no
    timing and failed at the header on a standard load. It is now generated in REAL TIME (lead-in + long
    leader, via the same engine as "→ WAV"), reading reliably on hardware/emulator. The .CAS stays
    fast-load (compact).

v1.0.26 — 2026-06-05
--------------------------------------------------------------------------------
NEW — "→ No loader (autostart)": SoftKristian tapes (opening screen + loader) become a BIN/DSK that RUNS
  • Tapes with a screen + tape-loader (e.g. QUASAR, STINGER, Zaxxon…) wouldn't run from disk: the
    multi-stage loader depends on the tape. The app now DETECTS the SoftKristian loader, extracts the
    addresses (screen → $0400, program → its original address, exec) and builds a .BIN/.DSK WITHOUT
    the loader:
      · the SCREEN goes straight to $0400; the PROGRAM stays at its ORIGINAL address (never relocated
