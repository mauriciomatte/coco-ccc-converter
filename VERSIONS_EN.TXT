v1.0.26 — 2026-06-05
--------------------------------------------------------------------------------
NEW — "→ No loader (autostart)": SoftKristian tapes (opening screen + loader) become a BIN/DSK that RUNS
  • Tapes with a screen + tape-loader (e.g. QUASAR, STINGER, Zaxxon…) wouldn't run from disk: the
    multi-stage loader depends on the tape. The app now DETECTS the SoftKristian loader, extracts the
    addresses (screen → $0400, program → its original address, exec) and builds a .BIN/.DSK WITHOUT
    the loader:
      · the SCREEN goes straight to $0400; the PROGRAM stays at its ORIGINAL address (never relocated
        — the 6809's absolute addressing would break); a short STUB reproduces the loader's "wait".
      · real AUTOSTART: writes a trap into the close vector $0176 — when LOADM finishes, the game
        STARTS BY ITSELF (no :EXEC), showing the opening screen. Just LOADM"NAME".
  • "→ No loader" button in the K7 tab: dialog with the detected addresses (editable), start mode
    (wait-key or ~10s delay) and target (.BIN or DSK pane). Warns if the program reaches the DOS
    stack (~$7E00) or the reserved RAM ($0600–$25FF).
  • Embedded "CoCo DCU" signature + "Revert" button: recognizes one of our files and recovers the
    PURE PROGRAM as .CAS/.WAV (dropping screen/stub) — round-trip conversion.
  • Validated on the 41-tape SoftKristian set: 23 recognized (19 full + 4 partial, flagged).
  • GUARD: the "↔ DSK" and "→ XRoar" buttons now DETECT a loader tape and warn/redirect to "→ No
    loader" (↔ DSK used to write the raw file at $009F = garbage that won't run).

NEW — K7 tab (datacorder)
  • "LINE/scope" waveform style: shows the SHAPE of the wave (the FSK square); the other styles
    (bars/peak/RMS) are amplitude envelopes. The .cas auto-zooms on open, showing the square wave.
  • "Default" button in Sound adjustments: resets to the safe default (Threshold 600µs, Amp 0, WAV 22kHz).
  • mini-XRoar is now a COLLAPSIBLE column: closed shows just a button (centered); opening runs it;
    closing turns it off. Size persists. (Frees the toolbar slot of the old "Mirror" button.)
  • New icons: → CAS (tape), → WAV (waves), ↔ DSK (floppy); → BASIC/→ XRoar/Undo/Redo are icon-only.

FIX — Tape export and reading
  • "→ Full tape" now has a 22 kHz FLOOR: at 11 kHz the 2400 Hz FSK corrupted (lost bytes → I/O ERROR
    on the CoCo). For a smaller, reliable WAV use the clean "→ WAV" (square wave).
  • "Extract → BIN" now produces a RUNNABLE DECB LOADM file (with load/exec header) — the raw payload won't run.
  • "(K8)" label removed from Sound adjustments and hints.

NEW — XRoar tab
  • ".bin AutoRun": opens a .bin/.rom as an XRoar boot parameter (xroar file.bin) and runs it automatically.
  • CTRL+ENTER = full reset (standardized with the mini-XRoar).
  • Fix: on emulator remount (machine/TV/boot change), the pending disk/tape is RE-MOUNTED on the new
    boot (it used to vanish from the emulator while the name stayed in the drive — you had to test 2×).

NEW — DSK tab
  • "Quick .BAS View" magnifier: read-only modal with the detokenized BASIC program.
  • Tape → DSK write: creates a blank disk if the pane is empty; confirms if a disk is already loaded.

v1.0.25 — 2026-06-04
--------------------------------------------------------------------------------
NEW — GAP-AWARE decoding (forensic analysis of screen/loader tapes)
  • Multi-stage loader tapes (2-part header → screen → program) were truncated at the 1st "gap":
    the app recovered only the loader and produced garbage. Now the tape is SEGMENTED by the
    pauses (tape hiss ≈ period < 330µs) and each segment is decoded in isolation and concatenated —
    recovering the WHOLE program (e.g. QUASAR 555 B → 4670 B; STINGER → 6715 B), noise-free.
  • The "pauses" aren't silence (the WAV is normalized): they're the MOTOR stopping while the
    loader draws the screen. Segmentation uses the PERIOD (frequency), not amplitude.

NEW — FAITHFUL multi-segment .CAS export that runs in XRoar
  • "→ CAS" and "Extract → CAS" rebuild the .cas preserving the per-segment structure (each
    segment with its own leader+sync and EOF) — CLOADM loads only the loader and IT reads the
    screen/program, like the tape. (Merging everything into one file broke the loader.)
  • IMPORTANT — .CAS vs .WAV and fast-load:
      · .CAS has NO timing (no pauses). It loads correctly with FAST-LOAD on (XRoar feeds the
        blocks abstractly) — that's the main XRoar tab's mode.
      · .WAV has the real gaps → loads correctly in REAL TIME (-no-tape-fast), the mini-XRoar's
        mode. That's why loader tapes run faithfully via the WAV/mini.
  • mini-XRoar gained a "Fast read" toggle: ON = test the .cas (fast-load, no animation); OFF =
    real-time WAV. Auto-on when opening a .cas file.
  • Opening a .cas shows the BASIC/Hex panels all at once (no reading animation).
  • Waveform editor: "Insert pause" button (1.0s silence at playhead/selection) to create/adjust
    the gaps between segments in the WAV (pauses only exist in the real-time .wav; .cas has none).

NEW — "Export" panel: kHz rate + REAL-TIME sizes
  • WAV rate selector (8/11/22/44 kHz) — lower = smaller file.
  • Live final size for → CAS, → WAV and → Full tape as you change rate/sound.
  • "→ Full tape" resamples the original audio (8-bit mono) to shrink while keeping everything.

NEW — Whole-tape Hex + collapsible Program panel
  • The Hex reveals the WHOLE program as the playhead passes each byte (not just the header).
  • "Program" panel collapsed by default (file + extension only); a highlighted "more info" button
    expands sizes/addresses/sync/Extract. K7 tab settings are persistent.

FIXED
  • "Extract → CAS" wouldn't open in XRoar (contiguous blocks without leaders) — now uses the
    canonical .cas. Normalize honors the selection; BASIC editor "save .cas" with inter-block leaders.

v1.0.12 — 2026-06-02
--------------------------------------------------------------------------------
ADDED (Save back into the MiniIDE .img image)
  • You can now EDIT a disk inside a MiniIDE image and SAVE it back into the .img (before it
    only saved a standalone .dsk). "Save" writes the current disk into its slot, in place.
  • Hardware-SAFE approach: the slot is sector-doubled (322,560 = 161,280 ×2). Instead of
    re-doubling blindly, we read the current slot and overwrite ONLY the "even chunks" (the
    256B CoCo sectors that de-double reads), PRESERVING the odd ones byte-for-byte — because a
    test on the REAL image showed the odd halves are NOT identical copies (CF leftovers/2nd
    copy). Only the sectors that actually changed are touched; the rest of the .img is intact.
  • Validated on the real MiniIDE image (507 MB, 337 disks): added file reads back with intact
    data, odd chunks preserved, neighbor disk untouched.
  • Switching disks in a container with pending edits writes the current disk back FIRST (no
    data loss). "Save As" of a file-container produces a standalone .dsk of the current disk.
  • RECOMMENDED to confirm on real hardware (write the .img to a CF and test on the MiniIDE).
